## Novedades de v2.2.0

### VIEW Directives (`cortex/v2/view.py`)

Implementa el sigilo `VIEW` como contrato declarativo de render reversible:

- **ViewDirective**: modelo con kind, target, reverse, fields, order, title, status, preserve
- **ViewKind**: 13 tipos (table, kv_table, matrix, list, numbered_list, checklist, prose, quote, puml, code, callout, section, raw)
- **ReverseStrategy**: 13 estrategias (rows_to_entries, row_to_attrs, verbatim_to_bloque, body_to_cuerpo, etc.)
- **KIND_REVERSE_COMPAT**: matriz de compatibilidad kind ↔ reverse
- **parse_view_entry()**: valida contrato mínimo (kind, target, reverse requeridos; kind+reverse compatibles)
- **resolve_target()**: resuelve selectores ($0:canonical_sigils, $0:contracts, SIGIL:*, SIGIL:name, $N, etc.)
- **calculate_view_coverage()**: calcula % de entries cubiertas por VIEW directives

### VIEW Renderer (`cortex/v2/view_renderer.py`)

Convierte CORTEX + VIEW → HCORTEX-R (Markdown reversible):

- Header con `reversible: true` y `view_schema: 1`
- `**Perfil: CORTEX-FULL**` como primera línea
- Cada VIEW produce un bloque con marcadores `<!-- VIEW:name ... -->` / `<!-- /VIEW:name -->`
- Renderizado por kind: tables (con headers canónicos), kv_tables, prose, quotes, PUML verbatim, lists, numbered lists, checklists
- Warnings por entries no cubiertas
- Reporte de coverage al final

### CLI `v2-convert --to hcortex-r`

```bash
cortex v2-convert SKILL.md --from cortex --to hcortex-r --out generated.md
```
