## Resumen

v2.3.0 implementa el salto fundacional: **CORTEX ⇄ HCORTEX verificable**,
no solo CORTEX → HCORTEX renderizable. Incluye parser inverso HCORTEX → AST,
encoder HCORTEX → CORTEX, motor de equivalencia (4 niveles), 11 errores
formales, 5 modos de operación, 7 comandos CLI nuevos, 12 tests de
aceptación T-01..T-12, y documentación obligatoria.

Cierra también los 8 prerrequisitos PRE-01..PRE-08 (en v2.2.3 intermedio).

## Limitaciones declaradas

- **AST-equivalence no perfecta:** El encoder reconstruye entries pero
  no preserva el orden exacto ni todos los tipos (attrs-pos, cuerpo
  multilínea). Esto se documenta como pérdida declarada, no silenciosa.
- **Hash verification estructural:** `hash` se parsea y preserva en el
  bloque, pero no se verifica contra el contenido. Pendiente v2.3.1.
- **Coverage por sigilo/sección/P-level:** Pendiente v2.3.1.
- **Modos audit/recovery:** Estructurales, no fully functional.
