## Brechas críticas cerradas (P0)

### P0-1 — recover reconstruye $0 para archivos entry-first

**Problema:** `recover` no detectaba "glosario vacío con entradas observables"
como equivalente a missing-glossary. Un archivo que empezaba con
`IDN:package{...}` se parseaba con un `$0` implícito vacío, y `recover` no
reconstruía los sigilos → `verify --strict` fallaba con `E003_UNKNOWN_SIGIL`.

**Fix:** `recover_cortex()` ahora verifica post-parse si `glossary.sigils`
está vacío mientras hay entradas observables, y dispara la reconstrucción.

**Verificación adversarial:**
```bash
echo 'IDN:package{name:"legacy"}
KNW:topic{topic:"x", content:"y", status:"current"}' > /tmp/legacy.cortex
cortex recover /tmp/legacy.cortex --out /tmp/fixed.cortex --embed-aud-rsk
cortex verify /tmp/fixed.cortex --strict  # rc=0, sin E003
```

### P0-2 — E031_SECRET_NOT_BYPASSABLE no es bypassable con --force

**Problema:** `--force` permitía persistir secretos en claro, contradiciendo
el SKILL §16.1 (`.cortex` MUST NOT almacenar secretos).

**Fix:**
- Nuevo código `E031_SECRET_NOT_BYPASSABLE` (reemplaza `E028`) con campo
  `bypassable=False`.
- `post_mutation_gate()` y `atomic_write_cortex()` respetan `bypassable=False`:
  ni `--force` ni `--no-validate-write` los sobrepasan.
- Nuevo flag `--unsafe-allow-secret-forensics` para bypass forense explícito
  de secretos (no de critical-sigil completeness).

**Verificación adversarial:**
```bash
cortex add brain.cortex --section $3 --sigil REF --name secret \
  --value 'password:"abc123456"' --force
# rc=1, E031_SECRET_NOT_BYPASSABLE, archivo sin cambios
```

### P0-3 — CRUD bloquea sigilos críticos incompletos por defecto

**Problema:** `W001_MISSING_FIELDS` era warning, permitiendo persistir
`OBJ:partial{goal:"partial"}` sin los campos requeridos.

**Fix:** Nuevo código `E032_CRITICAL_SIGIL_INCOMPLETE` (error,
`bypassable=False`) para sigilos críticos con campos faltantes. No es
bypassable ni con `--force`.

**Verificación adversarial:**
```bash
cortex add brain.cortex --section $2 --sigil OBJ --name partial \
  --value 'goal:"partial"'
# rc=1, E032_CRITICAL_SIGIL_INCOMPLETE
```

### P0-4 — FCS/OBJ status:done no cuenta como activo

**Problema:** Un brain con FCS y OBJ en `status:"done"` pasaba
`verify --strict --kind brain`. `done` es estado cerrado, no activo.

**Fix:** `active_status` cambiado de `{current, blocked, done}` a
`{current, blocked}`. Mensaje de error aclara que `done` es closed state.

**Verificación adversarial:**
```bash
cortex update brain.cortex FCS:primary --set status=done --force
cortex update brain.cortex OBJ:main --set status=done --force
cortex verify brain.cortex --strict --kind brain
# rc=1, E024_LEVEL2_MISSING_FOCUS (done is not active)
```

## Brechas medias cerradas (P1)

### P1-5 — Perfil como primera línea real de HCORTEX

**Problema:** HCORTEX-READ empezaba con `<!-- cortex-render: ... -->`, no
con `Perfil: CORTEX-<LEVEL>` como exige el SKILL §9.3.

**Fix:** `Perfil: CORTEX-<LEVEL>` ahora es la primera línea; el comentario
`cortex-render` va en la segunda. `edit_parser.py` actualizado para detectar
HCORTEX-READ/EDIT en las primeras 5 líneas.

### P1-6 — render --mode audit declara Mode: AUDIT

**Problema:** `render --mode audit` declaraba `Mode: READABLE` internamente.

**Fix:** `audit_mode_str` ahora refleja el modo real (`AUDIT` si audit_mode,
sino el modo del usuario).

### P1-7 — source visible en cuerpo/bloque

**Problema:** En AUDIT mode, `source` solo aparecía como columna en tablas
attrs; cuerpo y bloque solo tenían comentarios HTML.

**Fix:** `render_entry()` ahora añade una línea visible
`source: \`<SIGIL>:<name>\`` antes de bloques cuerpo/bloque/relación en
AUDIT mode.

## Brechas bajas cerradas (P2)

### P2-8 — cortex decode <file> con default usable

**Problema:** `cortex decode <file>` fallaba sin `--mode`.

**Fix:** `--mode` ahora es opcional con default `readable`. `decode` y
`render` funcionan sin `--mode`.

### P2-9 — Claim de recovery legacy rebajado

**Problema:** STATUS.md declaraba recovery legacy como `current` sin salvedad.

**Fix:** Cambiado a `current with known limits` con descripción de qué cubre
y qué no.

### P2-10 — Sincronizar canon SKILL externo e interno

**Problema:** Ambigüedad sobre la versión del SKILL (v1.1.0 vs v1.2.0).

**Fix:** STATUS.md ahora declara explícitamente que `SKILL_canon.md` es el
canon exacto (`v1.2.0-enterprise-candidate`) y que cualquier referencia
externa debe apuntar a ese archivo.

## Novedades de la v1.1.3

### Nuevos códigos de error (2)

- `E031_SECRET_NOT_BYPASSABLE` — secret scanning, `bypassable=False`
- `E032_CRITICAL_SIGIL_INCOMPLETE` — sigilos críticos incompletos, `bypassable=False`

### Cambios en API interna

- `recover_cortex()` detecta glosario vacío con entradas observables como
  equivalente a missing-glossary.
- `_scan_for_secrets()` produce `E031` con `bypassable=False`.
- `validate_level_policy()` produce `E032` para sigilos críticos incompletos.
- `validate_level_policy()` trata `done` como no-activo para FCS/OBJ.
- `post_mutation_gate()` y `atomic_write_cortex()` respetan `bypassable=False`.
- `atomic_write_cortex()` acepta `unsafe_allow_secret_forensics=True`.
- `render_hcortex_read()` pone `Perfil:` como primera línea real.
- `render_hcortex_read()` declara `Mode: AUDIT` correctamente.
- `render_entry()` añade `source:` visible para cuerpo/bloque en audit mode.

### Cambios en CLI

- `cortex add --unsafe-allow-secret-forensics` (forensic bypass for secrets only)
- `cortex decode <file>` y `cortex render <file>` funcionan sin `--mode`
  (default `readable`).

## Juicio técnico

La v1.1.3 cierra los 10 problemas detectados en la cuarta verificación
(`CODEC_CORTEX_VERIFICACION_v1_1_2_HCORTEX.md`). Los 4 P0 (críticos) cierran
bordes de gobernanza que podían contaminar memoria, seguridad y auditoría:

1. **recover entry-first** — el flujo de recuperación ahora funciona para
   archivos sin `$0` que empiezan directamente con entradas.
2. **secretos no-bypassable** — `--force` ya no puede persistir secretos en
   claro; se requiere `--unsafe-allow-secret-forensics` explícito para
   forensía.
3. **sigilos críticos incompletos** — `OBJ:partial{goal:"partial"}` ya no se
   persiste por defecto; los campos requeridos son obligatorios.
4. **done no es activo** — un brain con FCS/OBJ en `done` ya no pasa
   `verify --strict --kind brain`.

Los 3 P1 mejoran la auditoría HCORTEX (Perfil como primera línea real,
Mode:AUDIT declarado correctamente, source visible en cuerpo/bloque). Los
3 P2 cierran brechas de contrato CLI y documentación (decode sin --mode,
claim recovery rebajado, canon SKILL sincronizado).

La suite de tests creció de 124 → 147 (23 nuevos tests que verifican cada
fix adversarialmente, incluyendo casos CLI end-to-end). Todos los tests
pasan en venv limpia.
