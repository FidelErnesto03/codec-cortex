## Ajustes implementados (v2.2.1)

### P0-1: Eliminar HCORTEX-R como concepto canónico

- `--to hcortex-r` ahora emite: `WARNING: hcortex-r is deprecated. Use --to hcortex.`
- Header usa `internal_encoding: HCORTEX` (no HCORTEX-R)
- `render_hcortex_r` es alias de `render_hcortex` por compatibilidad

### P0-2: VIEW:view en $0 es sigil declaration, no directiva

- `parse_view_entries_from_doc()` ahora verifica: si VIEW está en $0 Y tiene `type/risk/cortex/desc`, es declaración de sigilo → se omite
- VIEW fuera de $0 sin kind/target/reverse → error

### P0-3: E_VIEW_* produce rc!=0

- `has_view_errors()` verifica si hay errores E_VIEW_*
- CLI `v2-convert` retorna rc=1 si hay errores VIEW

### P0-4: DIAG preserve:verbatim byte-identical

- `_render_puml()` y `_render_code()` no aplican `.strip()` cuando `preserve:verbatim`
- Blank lines internas se preservan

### P0-5: Header HCORTEX canónico

```
internal_encoding: HCORTEX
reversible: true
view_schema: 1
view_coverage: <N>
```

### P0-6: Marcadores VIEW con metadata completa

```html
<!-- VIEW:name kind=table target="..." reverse=rows_to_entries fields="..." order=source title="..." status=cur scope="..." preserve=verbatim hash="sha256:..." fallback="manual_review" -->
```

### P1: HUMAN_BLOCK compatible con prose

- `KIND_REVERSE_COMPAT[PROSE]` ahora incluye `PRESERVE_HUMAN_BLOCK`
- VIEW con `target:"HUMAN_BLOCK:..."` renderiza sin errores
