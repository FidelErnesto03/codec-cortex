# Informe de Entrega — codec-cortex v2.3.1

**Fecha:** 2026-06-30
**Versión:** 2.3.1 (corrective hard release)
**Autor:** Fidel Ernesto Lozada A.
**Licencia:** MIT
**Estado:** experimental (no enterprise-candidate)

## Resumen

v2.3.1 es un release correctivo post-auditoría de v2.3.0. El auditor
rechazó v2.3.0 como release bidireccional con justificación completa:
el roundtrip HCORTEX → CORTEX perdía estructura, VIEW directives, y
producía CORTEX inválido. v2.3.1 corrige los 8 P0 del auditor y
**reclasifica el estado real honestamente**.

## Estado real declarado

| Capacidad | Estado v2.3.0 | Estado v2.3.1 |
|---|---|---|
| CORTEX → HCORTEX | `current` (claim) | `current` (verificado) |
| HCORTEX → CORTEX | `current` (claim falso) | `experimental` (honesto) |
| Roundtrip bidireccional perfecto | `current` (claim falso) | `planned` (meta v2.3.2) |
| Tests que detectan fallos | `current` (claim falso) | `current` (4 tests en rojo) |

## P0 cerrados

### P0-1: Tests T-03/T-04/T-12 ahora fallan si la reversibilidad falla

- T-03 exige `AST-equivalent == True`
- T-04 exige `content-equivalent == True`
- T-12 ejecuta `v2-roundtrip-bidir` CLI y exige `rc == 0`
- **Resultado:** 4 tests ahora fallan honestamente, reflejando que el
  roundtrip bidireccional perfecto no se logra aún.

### P0-2: Sanitizar tablas HCORTEX → CORTEX

- `HEADER_MAP` con 40+ mapeos español→inglés (Sigilo→sigil, Tipo→type, etc.)
- `_strip_backticks()` elimina backticks de sigilos y valores
- `_sanitize_value()` normaliza whitespace

### P0-3: Reconstruir $13 con VIEW directives

- `_block_to_view_entry()` reconstruye cada VIEW desde el bloque parsed
- $13 ahora se reconstruye con 44 VIEW directives (antes: 0)

### P0-4: Fix v2-roundtrip-bidir Direction 2

- Lógica corregida: ya no usa CORTEX como si fuera HCORTEX
- CORTEX input → Direction 1 (C→H→C) + Direction 2 (C→H→C→H)
- HCORTEX input → Direction 1 (H→C→H) + Direction 2 (H→C→H→C)

### P0-5: Coverage 0 entries + 0 directives = 0%

- `calculate_view_coverage()` ya no retorna 1.0 para docs vacíos
- Si `len(eligible) == 0`, retorna 0.0

### P0-6: Post-write validation

- `encode_cortex_from_ast()` re-serializa y reparsa
- Si `re_entries < declared_entries`, emite `E_AST_EQUIVALENCE_FAIL`
- CLI `v2-convert --from hcortex` no escribe `--out` si hay errores

### P0-7: E_VIEW_HASH_MISMATCH real

- `_compute_block_hash()` calcula SHA-256 del contenido del bloque
- Si no coincide con el hash declarado, emite `E_VIEW_HASH_MISMATCH`

### P0-8: Limpiar .pytest_cache del paquete

- Script de empaquetado elimina caches antes de crear tarball

## Limitaciones declaradas (honestas)

### Roundtrip bidireccional NO perfecto

El encoder reconstruye **182/266 entries (68%)**. Pierde entries cuando:

1. **Tablas sin columna `name`:** Usan contenido de `rule` como nombre,
   produciendo nombres largos no reparsables (ej: `CNST:FCS requiere what...`)
2. **DIAG bloques multilínea ($6):** PUML con `@startuml/@enduml` no se
   reconstruyen correctamente
3. **KNW profiles ($9):** Tablas con campos `topic/content/status` se
   reconstruyen pero con nombres sintéticos que no matchean originales

### 4 tests en rojo intencionalmente

| Test | Razón del fallo |
|---|---|
| T-03 | AST-equivalent == False (248 diffs) |
| T-04 | content-equivalent == False (33 diffs) |
| T-12 | v2-roundtrip-bidir rc=1 |
| test_cli_v2_convert_hcortex_to_cortex | E_AST_EQUIVALENCE_FAIL |

Estos tests **deben** fallar hasta que el roundtrip bidireccional
perfecto se logre en v2.3.2.
