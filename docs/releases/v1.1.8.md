## Fixes cerrados (v1.1.8)

### Fix 1 — Recovery elige sección libre real

recover ahora escanea $1, $2, ..., $99, $100, ... hasta encontrar la primera sección verdaderamente libre. Nunca contamina una sección existente, incluso si $1 y $99 ya existen.

### Fix 2 — --embed-aud-rsk embebe RSK para estado vivo recuperado

`_embed_recovery_trace()` ahora inserta `RSK:recovered_live_*` para cada sigilo de estado vivo (FCS/OBJ/WRK/STP/NXT) movido desde $0, además del W011_RECOVERED_LIVE_STATE diagnóstico.

### Fix 3 — AUD describe el evento real

AUD:recovery ahora describe el evento real:
- `glossary_reconstruction` si $0 fue reconstruido
- `live_state_recovered_from_zero` si se movió estado vivo
- Ambos combinados si ambos ocurrieron

Ya no dice siempre `glossary_reconstruction` cuando solo se movieron entradas.

### Fix 4 — E034 cubre null-like sentinels

`_is_field_empty()` ahora detecta: `null`, `none`, `nil`, `undefined`, `n/a`, `n.a.`, `na`, `tbd`, `todo`, `tbc`, `???`, `?`, `-`, `--` (todos case-insensitive). El parser de attrs y el CLI `--set` también convierten `none`/`nil`/`undefined` a Python None.
