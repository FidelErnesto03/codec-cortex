## Brechas cerradas (v1.1.4)

### P0-1 — E024 non-bypassable: --force no puede degradar cerebro operativo

**Problema:** `cortex delete brain.cortex FCS:primary --force` persistía un brain
sin FCS activo. El CLI detectaba el error (E024) pero lo escribía igual porque
`--force` lo sobrepasaba.

**Fix:** Todas las invariantes de gobernanza cognitiva (E023, E024, E026, E029)
ahora tienen `bypassable=False`. Ni `--force` ni `--no-validate-write` pueden
sobrepasarlas. El `post_mutation_gate` bloquea la escritura **antes** de tocar
el archivo — el archivo en disco queda intacto.

**Verificación:**
```bash
cortex delete brain.cortex FCS:primary --force  # rc=1, archivo sin cambios
cortex verify brain.cortex --strict --kind brain  # sigue pasando
```

### P0-2 — --force no puede eliminar P0 operativo

Cubierto por P0-1: las invariantes E023/E024/E026/E029 son non-bypassable, por
lo que `--force` no puede eliminar FCS, OBJ, CNST:blocking ni STP si eso rompe
las invariantes del cerebro operativo.

### P0-3 — --unsafe-allow-secret-forensics marca non_conformant

**Problema:** El flag forense escribía un .cortex con secreto y respondía
`ok:true` sin marcar el artefacto como no conforme.

**Fix:** Cuando se usa `--unsafe-allow-secret-forensics`, se añade una entrada
`STAT:forensic_quarantine` con `conformity:"non_conformant_forensic_artifact"` y
`status:"deprecated"`. El JSON de respuesta incluye un `warning` explícito.
`verify --strict` sigue fallando sobre el artefacto forense.

### P1-4 — pytest como dependencia dev

Añadido `[project.optional-dependencies] dev = ["pytest>=7.0"]` en
`pyproject.toml`. Instrucciones: `pip install -e ".[dev]"`.

### P1-5 — Demo v1.1.4

Creado `scripts/cortex_demo_v1_1_4.sh` — portable, reproducible, exit 1 real
en fallo. Prueba los 6 fixes de v1.1.4.

### P1-6 — RSK general cuando recover reconstruye $0

**Problema:** recover solo añadía RSK para sigilos no canónicos. Si todos los
sigilos eran canónicos, no quedaba RSK que marcara el riesgo de la
reconstrucción.

**Fix:** recover ahora añade siempre `RSK:reconstructed_glossary` cuando el $0
fue reconstruido, incluso si todos los sigilos son canónicos.

## Invariantes non-bypassable (v1.1.4)

| Código | Descripción | bypassable |
|---|---|---|
| E023_LEVEL1_LIVE_STATE | Nivel 1 con estado vivo | `False` |
| E024_LEVEL2_MISSING_FOCUS | Nivel 2 sin FCS/OBJ activo | `False` |
| E026_BLOCKING_NOT_P0 | CNST:blocking sin survive:min | `False` |
| E029_LEVEL3_LIVE_STATE | Nivel 3 con estado vivo | `False` |
| E031_SECRET_NOT_BYPASSABLE | Secreto en claro | `False` |
| E032_CRITICAL_SIGIL_INCOMPLETE | Sigilo crítico incompleto | `False` |

Estas 6 invariantes **no pueden ser sobrepasadas** por `--force` ni
`--no-validate-write`. Solo `--unsafe-allow-secret-forensics` puede sobrepasar
E031 (forensic recovery explícito), y el artefacto resultante se marca como
`non_conformant_forensic_artifact`.
