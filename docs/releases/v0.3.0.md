## Novedades de la v1.1.0

### Nuevos módulos

- `cortex/core/document_kind.py` — `DocumentKind`, `infer_document_kind()`, `validate_level_policy()` (gobierno cognitivo)
- `cortex/hcortex/profiles.py` — `Profile`, `PROFILES`, `classify_entry()`, `filter_by_profile()`, `sort_by_plevel()`
- `cortex/hcortex/recovery.py` — `recover_cortex()`, `strip_preamble()`, `normalise_legacy_type_name()`
- `cortex/cli/commands/recover.py` — comando `cortex recover`
- `cortex/cli/commands/diagram.py` — comando `cortex diagram list/extract/validate`

### Nuevos códigos de error (8)

`E023_LEVEL1_LIVE_STATE`, `E024_LEVEL2_MISSING_FOCUS`, `E025_INVALID_SURVIVE`,
`E026_BLOCKING_NOT_P0`, `E027_ATTRS_POS_ARITY`, `E028_SECRET_IN_CLEAR`,
`E029_LEVEL3_LIVE_STATE`, `E030_RECOVERY_INCOMPLETE`.

### Nuevos flags CLI

- `verify --strict` (warnings → errors)
- `verify --kind brain|skill|package|generic` (kind explícito)
- `verify --roundtrip cortex` (roundtrip .cortex → .cortex, sin HCORTEX)
- `doctor --strict --kind <kind>`
- `diff --profile structural|semantic|governance`
- `render --mode readable|audit|recovery|full`
- `render --profile min|recovery|work|full`

### Nuevos comandos CLI

- `cortex recover <file> --out <file>` — recuperación de artefactos legacy
- `cortex diagram list <file>` — listar entradas DIAG
- `cortex diagram extract <file> --name <n> --out <file>` — extraer bloque verbatim
- `cortex diagram validate <file> [--name <n>]` — validar integridad PlantUML

## Juicio técnico

La v1.1.0 cierra todas las brechas de auditoría identificadas en
`CODEC_CORTEX_CLI_AUDIT_HCORTEX.md` y añade tests explícitos (G1–G8)
que las protegen contra regresiones. El CLI pasa de "subconjunto
funcional permisivo" a "codec gobernado con disciplina cognitiva":

- Valida separación de niveles (Nivel 1 sin estado vivo, Nivel 2 con
  FCS+OBJ, Nivel 3 sin WRK).
- HCORTEX-READ ahora es canónico: declara `Perfil: CORTEX-<LEVEL>`,
  filtra por P-level y ordena P0→P5.
- El modo `AUDIT` agrega columna `source` para trazabilidad.
- El bug de pipes en tablas Markdown está cerrado.
- Los artefactos legacy pueden recuperarse con `cortex recover`.
- El README ya no sobredeclara capacidades; `STATUS.md` y `BENCHMARK.md`
  documentan explícitamente qué es `current` vs `planned` vs `future`.

Per la taxonomía del SKILL §0.3, las capacidades de la v1.1.0 pueden
declararse como `current` con la evidencia reproducible en
`BENCHMARK.md`. Las capacidades `planned`/`future` (MCP server, runtime
de maduración, benchmark adversarial amplio) siguen declaradas como
tales en `STATUS.md`.
