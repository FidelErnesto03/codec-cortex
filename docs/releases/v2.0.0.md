## Qué se implementó en v2.0.0

### Módulo `cortex.v2` — Parser/Writer CORTEX v2

Nuevo módulo que maneja el formato CORTEX exacto usado en `skill/cortex/SKILL.md`:

1. **Parser CORTEX v2** (`cortex/v2/parser.py`):
   - Wrapper `markdown` (apertura y cierre)
   - Header `<!-- CODEC-CORTEX -->` con `internal_encoding`, `source_artifact`, etc.
   - Secciones `$0`–`$12` (bare `$N` en línea propia)
   - `$0` basado en entries: `IDN:identity{type:attrs,risk:B,cortex:Semantic,desc:"..."}`
   - `$0` metadata: `$0:type_attrs{rule:"..."}`, `$0:contract_hdl{pos:"..."}`, `$0:micro_cur{expand:current}`
   - HDL bare `attrs-pos`: `HDL:agent_init|specification|SKILL.md|notes...` (sin llaves)
   - Micro-tokens como bare values: `status:cur`, `survive:min`, `risk:B`
   - `DESC`/`AXM` (cuerpo): `SIGIL:name{long text...}`
   - `DIAG` (bloque): `SIGIL:name{` + multiline content + `}`
   - `!` entries: `!name{rule:"...",survive:min}` (sin `:` después de `!`)
   - `!:rule{type:attrs,...}` en `$0` (sigil declaration con `:`)

2. **Writer CORTEX v2** (`cortex/v2/writer.py`):
   - Reproduce el formato exacto byte-por-byte
   - Reglas de quoting: bare para enum/micro, quoted para texto libre
   - `cortex:"Episodic/Visual"` quoted cuando contiene `/`
   - `pos:"operation|status|requires|notes"` quoted cuando contiene `|`
   - HDL sin trailing `|` para campos vacíos
   - `!` entries sin `:` en $4+, con `:` en $0 (sigil declaration)

3. **CLI command** `cortex v2-roundtrip <file>`:
   - Verifica que CORTEX → parse → write produce output byte-identical
   - Reporta secciones, entries, bytes y primera diferencia si falla

### Roundtrip verificado

```
cortex v2-roundtrip SKILL.md
✓ Roundtrip byte-identical: SKILL.md
  sections: 13
  entries:  222
  bytes:    37399
```

El archivo `SKILL.md` (388 líneas, 37399 bytes) se parsea a un AST de 13 secciones y 222 entries, y se serializa de vuelta produciendo **exactamente los mismos bytes**.

## Pendiente para v2.1.0

La traducción CORTEX → HCORTEX (y viceversa) requiere un renderer/parser HCORTEX v2 que maneje el formato rico de `SKILL_HCORTEX.md` (2074 líneas con tablas, PUML, prose, cross-references). El análisis técnico reveló que:

1. **HCORTEX tiene 15 secciones adicionales** (§12–§26) que no existen en CORTEX
2. **Las entradas se reestructuran**: HDL → PUML + prose, KNW → tablas, DESC → párrafos
3. **Se añade contexto humano**: cross-references, ejemplos, plantillas, checklists
4. **No es una transformación mecánica**: requiere juicio sobre cómo decomponer entradas densas en formato humano legible

Por estas razones, la traducción HCORTEX exacta es un proyecto mayor que se planea para v2.1.0.
