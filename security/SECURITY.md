# Security Policy

## Supported Versions
El estándar y la implementación de referencia siguen versionado independiente
(see `governance/versioning.md` cuando exista).

## Reporting
Reportar vulnerabilidades de forma privada al mantenedor. No abrir issues públicos
con detalles de explotación.

## Threat Model
Ver `security/THREAT_MODEL.md` (pendiente de Fase 1).
