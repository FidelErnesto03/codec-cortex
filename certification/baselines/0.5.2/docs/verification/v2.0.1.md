## 4. Brechas principales

### H-01 — No existe conversión CORTEX → HCORTEX canónica

| Campo | Valor |
|---|---|
| Severidad | Alta |
| Evidencia | `cortex render /mnt/data/SKILL.md` genera un HCORTEX-READ no equivalente a `SKILL_HCORTEX.md` |
| Impacto | No cumple la intención de producir `SKILL_HCORTEX.md` desde `SKILL.md` |
| Requisito | Renderer HCORTEX v2 basado en IR, no renderer heredado |

### H-02 — No existe conversión HCORTEX → CORTEX canónica

| Campo | Valor |
|---|---|
| Severidad | Alta |
| Evidencia | `cortex compile /mnt/data/SKILL_HCORTEX.md` falla con `E011_HCORTEX_EDIT_METADATA_MISSING` |
| Impacto | No puede producir `SKILL.md` desde `SKILL_HCORTEX.md` |
| Requisito | Parser HCORTEX v2 que entienda header, tablas, PUML, prose, listas, checklists y bloques editoriales |

### H-03 — No existe IR común

| Campo | Valor |
|---|---|
| Severidad | Alta |
| Evidencia | Solo existen `cortex.v2.parser` y `cortex.v2.writer`; no hay módulos `convert`, `ir`, `bidir`, `compare-hcortex` ni `roundtrip-bidir` |
| Impacto | No hay lugar donde preservar simultáneamente entradas CORTEX y bloques humanos HCORTEX |
| Requisito | `SkillIR` o equivalente: entradas CORTEX + bloques HCORTEX + mapping + checksums + warnings |

### H-04 — El parser viejo procesa mal `SKILL.md` CORTEX v2

| Campo | Valor |
|---|---|
| Severidad | Alta |
| Evidencia | `cortex verify /mnt/data/SKILL.md --strict` produce 390 errores; `render` advierte que `$0` contiene entradas operativas |
| Impacto | El CLI tiene dos mundos no integrados: parser antiguo y parser v2 |

## 5. Recomendación para v2.1.0

### 5.1 No intentar convertir directo con regex

La conversión debe pasar por un IR común:

```text
SKILL.md CORTEX
        ↓ parse_cortex_v2
      SkillIR
        ↓ render_hcortex_v2
SKILL_HCORTEX.md
        ↓ parse_hcortex_v2
      SkillIR
        ↓ write_cortex_v2
SKILL.md CORTEX
```

### 5.2 Modelo mínimo de IR

| Entidad | Debe conservar |
|---|---|
| `Header` | `internal_encoding`, `source_artifact`, `source_version`, `status`, campos extra |
| `CortexEntry` | sigil, name, type, attrs/body/block, section, order, raw, checksum |
| `HCortexBlock` | heading path, kind(table/prose/puml/list/checklist/code), raw markdown, source mapping, checksum |
| `Mapping` | `CORTEX_ENTRY ↔ HCORTEX_BLOCK(S)`; many-to-one y one-to-many |
| `HumanBlock` | bloques HCORTEX no derivables desde CORTEX |
| `Warning` | contenido huérfano, duplicado, ambiguo, no mapeado |

### 5.3 Comandos esperados

```bash
cortex v2-convert --from cortex --to hcortex SKILL.md --out generated_HCORTEX.md
cortex v2-convert --from hcortex --to cortex SKILL_HCORTEX.md --out generated_CORTEX.md
cortex v2-compare --kind cortex generated_CORTEX.md SKILL.md
cortex v2-compare --kind hcortex generated_HCORTEX.md SKILL_HCORTEX.md
cortex v2-roundtrip-bidir --cortex SKILL.md --hcortex SKILL_HCORTEX.md
```

### 5.4 Gates enterprise mínimos

| Gate | Criterio |
|---|---|
| G1 | `SKILL.md → HCORTEX` genera archivo content-equivalent a `SKILL_HCORTEX.md` |
| G2 | `SKILL_HCORTEX.md → CORTEX` genera archivo AST-equivalent a `SKILL.md` |
| G3 | Todo sigilo CORTEX tiene representación HCORTEX o mapping explícito |
| G4 | Toda tabla HCORTEX se mapea a entrada/grupo/bloque IR o `HumanBlock` |
| G5 | Todo PUML conserva contenido verbatim |
| G6 | Todo bloque humano no derivable queda preservado, no descartado |
| G7 | No hay contenido huérfano sin warning |
| G8 | Comparadores distinguen `byte-identical`, `AST-equivalent`, `semantic-equivalent`, `content-equivalent` |
| G9 | Router `internal_encoding` evita usar parser antiguo con CORTEX v2 |
| G10 | Benchmarks documentan pérdidas, omisiones y equivalencias con evidencia reproducible |
