### 4.2 Integración en `cortex doctor`


## 5. E2.3 — Mutation gates

### 5.1 Nuevo módulo `cli/src/cortex/core/modes.py`

```python
class Mode(str, Enum):
    READ_ONLY = "read-only"
    EDITOR = "editor"      # default
    ADMIN = "admin"
```

### 5.2 Resolución de modo

Prioridad: `--mode` flag > `$CORTEX_MODE` env var > default (`editor`).

### 5.3 Comportamiento

| Modo | Writes | --force | Exit code |
|------|:------:|:-------:|:---------:|
| read-only | Bloqueado | Bloqueado | 13 (E035) |
| editor (default) | Permitido | Prompt interactivo; CI procede sin prompt | 0/1 normal |
| admin | Permitido | Permitido | 0/1 normal |

### 5.4 Flags añadidos a `main.py`

```bash
cortex --mode read-only delete brain.cortex FCS:primary     # rc=13
cortex --mode editor delete brain.cortex FCS:primary --force # prompt y/N
cortex --mode admin delete brain.cortex FCS:primary --force  # sin prompt
cortex --yes --mode editor delete brain.cortex FCS:primary --force  # pre-confirmado
CORTEX_MODE=read-only cortex delete brain.cortex FCS:primary # rc=13
```

> **Nota:** el `dest` del flag global es `op_mode` (no `mode`) para no
> colisionar con el `--mode` existente de `v2-convert` (que tiene
> choices `normal/strict/audit/recovery/display`).

## 6. E2.4 — Audit log bajo demanda

### 6.1 Nuevo módulo `cli/src/cortex/audit/logger.py`

- **Sin logging automático.** Ninguna operación se loguea por defecto.
- `cortex audit on` crea `~/.codec-cortex/audit/session.state`.
- Las mutaciones CRUD se loguean a `~/.codec-cortex/audit/YYYY-MM-DD.jsonl`.
- `cortex audit off` elimina el state file (preserva logs existentes).
- `cortex audit snapshot <file>` exporta copia verbatim a `~/.codec-cortex/audit/snapshots/`.
- `cortex audit prune --keep-days 30` elimina logs viejos.
- Rotación automática a `.jsonl.1` si el archivo supera 10 MB.
- Override via `$CORTEX_AUDIT_DIR` (para tests).

### 6.2 Formato JSONL

```json
{"t":"2026-07-01T12:00:00Z","op":"add","file":"brain.cortex","mode":"editor","result":"ok","selector":"FCS:primary"}
```

### 6.3 Logging automático en `main.py`

Después de cada mutación (add/update/delete/move/format/compile/recover), si
logging está ON, se llama `append_entry()` con el resultado. Los comandos
de lectura (list/get/verify/...) NO se loguean.

## 8. E2.6 — `cortex verify --signature`

### 8.1 Nuevo módulo `cli/src/cortex/security/signature.py`

API: `parse_manifest(path)`, `verify_signature(target, manifest, strict)`,
`SignatureResult(ok, file, expected_hash, actual_hash, reason)`.

### 8.2 Uso

```bash
# Verificar un artefacto contra SHA256SUMS
cortex verify brain.cortex --signature SHA256SUMS
# → ok=True, rc=0 (match)
# → ok=False, rc=1 (mismatch/missing)

# Alias --manifest
cortex verify brain.cortex --manifest SHA256SUMS

# Modo strict: falla si el archivo no está en el manifest
cortex verify brain.cortex --signature SHA256SUMS --strict
```

### 8.3 Reasons

| Reason | Significado | ok |
|--------|-------------|:--:|
| `match` | Hash coincide | True |
| `mismatch` | Hash no coincide | False |
| `missing_in_manifest` | Archivo no listado | False |
| `missing_file` | Archivo no existe | False |
| `malformed_manifest` | Manifest vacío/malformado | False |
| `no_manifest` | Manifest no existe | False |

## 9. Tests (68 nuevos)

### 9.1 `test_e2_signature.py` (12 tests)

- `sha256_file`: hashing de archivos pequeños y grandes (streaming).
- `parse_manifest`: formato estándar, comentarios, star-prefix, missing file.
- `verify_signature`: match, mismatch, missing_in_manifest, no_manifest, missing_file.
- CLI: `cortex verify --signature` match/mismatch/missing.

### 9.2 `test_e2_secret_scanner.py` (17 tests)

- Pattern detection: ghp_, pypi-, AKIA, PEM, password, slack.
- Severity classification: high vs medium.
- File scanning: binary skip, UTF-8 read.
- Directory walking: recursive, hidden dirs skip.
- Baseline suppression.
- Redaction.

### 9.3 `test_e2_modes.py` (16 tests)

- `Mode` enum: from_string defaults, aliases, unknown raises.
- Properties: allows_writes, allows_force.
- Command classification: is_read/is_write/is_meta.
- `resolve_mode`: CLI flag > env var > default.
- `check_permission`: read-only blocks, editor allows, admin allows, non-interactive proceeds.
- CLI: `--mode read-only` blocks delete/add, allows list; `$CORTEX_MODE` works; `--mode admin` allows force.

### 9.4 `test_e2_audit.py` (23 tests)

- Session state: disabled by default, enable creates file, disable removes file.
- `append_entry`: nothing written when disabled, writes line when enabled, appends multiple.
- `AuditEntry`: JSONL single-line, error_code included.
- `snapshot_file`: copies content, label in filename, missing source raises.
- CLI: `audit status/on/off/snapshot`, mutation logged when on, not logged when off.
