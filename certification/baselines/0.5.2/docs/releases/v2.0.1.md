## Brechas cerradas (v2.0.1 vs v2.0.0)

### P0-1: Rebaseado sobre v1.1.9 real

v2.0.0 estaba basado en v1.1.7 + módulo v2. v2.0.1 incorpora todos los fixes de v1.1.8 y v1.1.9:

- **v1.1.8 Fix 1:** Recovery elige sección libre real (scan $1, $2, ..., sin techo)
- **v1.1.8 Fix 2:** `--embed-aud-rsk` embebe RSK para `W011_RECOVERED_LIVE_STATE`
- **v1.1.8 Fix 3:** AUD describe el evento real (`live_state_recovered_from_zero`, no siempre `glossary_reconstruction`)
- **v1.1.8 Fix 4:** E034 cubre null-like sentinels (`none`, `nil`, `undefined`, `n/a`, `tbd`, `???`, `-`, `--`)
- **v1.1.9 Fix 1:** `recover` repara `$0` existente pero incompleto (`_repair_incomplete_glossary`)
- **v1.1.9 Fix 2:** `--embed-aud-rsk` traza reparación de `$0` incompleto (`AUD:incomplete_glossary_repair`, `RSK:auto_declared_*`)
- **v1.1.9 Fix 3:** Sección libre sin techo artificial (`_first_free_recovery_section_id`)

### P0-2: v2-roundtrip compara bytes reales

v2.0.0 comparaba strings leídos en modo texto. v2.0.1 usa `open(path, "rb")` y compara `bytes`:

```python
with open(args.input, "rb") as f:
    original_bytes = f.read()
# ... parse and write ...
reproduced_bytes = reproduced_text.encode("utf-8")
assert original_bytes == reproduced_bytes
```

### P0-3: Reporta bytes reales

v2.0.0 reportaba `len(str)` como "bytes". v2.0.1 reporta:
- `stat_bytes`: `os.path.getsize(path)` — tamaño real del archivo
- `text_chars`: `len(text)` — caracteres Unicode
- `repro_bytes`: `len(reproduced_bytes)` — bytes del output reproducido

```
✓ Roundtrip byte-identical: SKILL.md
  stat_bytes: 38018
  text_chars: 37399
  repro_bytes: 38018
```

### P1-4: Claim corregido a "byte-identical"

El claim ahora es técnicamente correcto: compara `bytes` reales, no `str`.
