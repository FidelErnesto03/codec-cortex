## Novedades de v2.1.0

### SkillIR — Intermediate Representation

Nuevo módulo `cortex/v2/ir.py` que define el modelo intermedio compartido:

- `SkillIR`: contenedor con entries, blocks, warnings, section_map
- `IREntry`: entrada CORTEX con mapping a blocks HCORTEX
- `IRBlock`: bloque HCORTEX con mapping a entries CORTEX
- `IRWarning`: advertencias de contenido no mapeable

Flujo de conversión:
```
CORTEX → parse_cortex_v2 → CortexV2Document → cortex_to_ir → SkillIR
SkillIR → render_hcortex_v2 → HCORTEX Markdown
SkillIR → ir_to_cortex → CortexV2Document → write_cortex_v2 → CORTEX
```

### HCORTEX v2 Renderer

Nuevo módulo `cortex/v2/hcortex_renderer.py` que convierte SkillIR a Markdown humano:

- Header con `internal_encoding: HCORTEX`
- `**Perfil: CORTEX-FULL**` como primera línea de contenido
- Secciones numeradas (# N. Title)
- Tablas para sigilos, tipos, contratos, micro-tokens, constraints, pitfalls
- PUML blocks para DIAG entries (verbatim)
- Prose para DESC/AXM entries
- Handlers con operation/status/requires/notes

### CLI `v2-convert`

```bash
cortex v2-convert SKILL.md --from cortex --to hcortex --out generated.md
```

### M-01 Fix: Header notes quoting

El parser ahora detecta valores quoted en el header (`notes: "texto con espacios"`) y los desempaqueta. El writer los vuelve a quoting cuando contienen espacios, comas o caracteres especiales.
