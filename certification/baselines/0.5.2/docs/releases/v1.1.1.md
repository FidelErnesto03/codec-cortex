## Novedades de la v1.1.1

### Cambios en API interna

- `cortex.core.validator.validate(doc, strict=False, kind=None)` — acepta `kind`
- `cortex.core.validator.is_valid(doc, strict=False, kind=None)` — acepta `kind`
- `cortex.core.document_kind.LIVE_WORKING_SIGILS` — `{FCS, OBJ, WRK, STP, NXT}`
- `cortex.core.document_kind.LIVE_SESSION_SIGILS` — `{SES, LNG}`
- `cortex.core.document_kind.LIVE_STATE_SIGILS` — unión de los dos anteriores
- `cortex.core.document_kind.SKILL_FORBIDDEN_LIVE_SIGILS` — alias
- `cortex.core.writer.serialize_attrs_pos()` — levanta `InvalidValueError` si valor contiene `|`
- `cortex.crud.mutations.add_entry(..., allow_unknown_sigil=False)` — bloquea sigilos desconocidos por defecto
- `cortex.hcortex.read_renderer.render_hcortex_read(..., layout=None)` — soporta `priority`/`section`
- `cortex.hcortex.recovery.recover_cortex(..., embed_aud_rsk=False)` — inserta AUD/RSK opcional

### Cambios en CLI

- `cortex verify --kind <kind>` ahora gobierna validación (no solo display)
- `cortex doctor --kind <kind>` ahora gobierna validación
- `cortex render --layout priority|section` (auto-selección por defecto)
- `cortex add --allow-unknown-sigil`, `--no-validate-write`, `--strict-write`
- `cortex diagram extract --print-newline` (stdout friendly)
- `cortex diff --profile governance --format json` retorna non-zero con findings
- `cortex recover --embed-aud-rsk`

### Gobernanza de madurez

- `STATUS.md` actualizado con matriz v1.1.1 (37 capacidades clasificadas)
- `BENCHMARK.md` reclasificado (throughput: `measured` → `hypothesis`)
- `CHANGELOG.md` con entrada detallada v1.1.1
- `SKILL_canon.md` en el tarball
- `scripts/` con 3 demos (v1.0, v1.1, v1.1.1)
- `AUDIT_BASE.md` y `AUDIT_REAUDIT_BASE.md` como trazabilidad

## Juicio técnico

La v1.1.1 cierra todas las brechas de la re-auditoría (`CODEC_CORTEX_REAUDIT_v1_1_0_HCORTEX.md`)
con tests CLI end-to-end que verifican el comportamiento real del CLI (no solo
la API interna). Los hallazgos más críticos estaban en:

1. **`--kind` no gobernaba la validación** (H-RA-01) — el kind explícito se
   creaba pero se descartaba al llamar `validate()`. Ahora se propaga
   correctamente.

2. **SES/LNG no se bloqueaban en skills** (H-RA-02) — la lista
   `LIVE_STATE_SIGILS` solo contenía working sigils. Ahora incluye session
   sigils, con excepción para entradas marcadas como ejemplo/contrato.

3. **`attrs-pos` perdía silenciosamente valores con `|`** (H-RA-05) — el
   parser dividía por `|` sin respetar comillas. Ahora el writer rechaza
   explícitamente pipes en valores attrs-pos (opción A del audit).

4. **CRUD permitía persistir sigilos no declarados** (H-RA-06) — `add_entry`
   no validaba contra `$0`. Ahora levanta `UnknownSigilError` por defecto.

5. **`diagram extract` añadía newline final** (H-RA-04) — rompía la
   promesa de verbatim. Ahora escribe bytes exactos.

La suite de tests creció de 88 → 110 (22 nuevos tests CLI end-to-end que
verifican cada fix desde el shell, no solo desde Python). Todos los tests
pasan en venv limpia.
