# BLP-010: Redefinicion HCORTEX — esquemas explicitos
<!-- /BLP:TITLE -->

---

<!-- BLP:1 -->
## §1: Planteamiento del Problema

La Fase 4 actual define HCORTEX como markdown sin schema explícito. Esto obliga al parser de HCORTEX a inferir estructura desde markdown genérico (tablas por pipes, fences por backticks), lo que introduce ambigüedad en el roundtrip CORTEX↔HCORTEX.

Además, la visión del estándar ha madurado: CORTEX es un lenguaje de máquina para modelos de lenguaje. HCORTEX es su representación humana. El roundtrip no debe depender de inferencia de markdown — debe ser directo, determinista, basado en esquemas explícitos.

**Evidencia:**
- La Fase 4 actual produce HCORTEX markdown genérico que requiere inferencia para compilar de vuelta a CORTEX
- El primer revisor externo señaló ambigüedad en el roundtrip (B-006: igualdad lógica AST sub-especificada)
- El conjunto de esquemas visuales es finito y cerrado (table, puml, code, body, list, rel) — no necesita extensión dinámica
- El paralelo con $0 en CORTEX es exacto: así como $0 declara contratos para el parser de CORTEX, los schemas declararían visualización para el parser de HCORTEX

**Impacto de no resolverlo:**
- HCORTEX seguirá siendo markdown ambiguo → roundtrip frágil
- El parser de HCORTEX requerirá inferencia → complejidad innecesaria
- El Gate F4 no podrá certificarse limpiamente
<!-- /BLP:1 -->

<!-- BLP:2 -->
## §2: Objetivo

Redefinir HCORTEX como markdown con esquemas explícitos (`<!-- schema=... -->`), documentar el conjunto cerrado de esquemas (table, puml, code, body, list, rel), definir las reglas de transformación HCORTEX↔CORTEX para cada esquema, y actualizar la especificación hcortex-0.1.md. Sin código. Solo especificación.

## §8: Diseño Técnico

9. **Encabezado HCORTEX.** Todo archivo HCORTEX comienza con `<!-- HCORTEX v=0.1 t=canonical k=brain -->` donde v=versión del formato (obligatorio), t=tipo de render: canonical (reversible) o readable (no reversible), k=kind del documento (opcional: brain, mission, inventory, etc.). El roundtrip mapea contra `$0:format{cortex:0.1,encoding:UTF-8,type:brain}` en CORTEX.
