# BLP-001: Robustecer el parser Python oficial (codec_cortex/parser.py) para alcanzar 40/40 en el corpus F4, tomando como referencia la implementacion Go que ya procesa el 100% de los casos
<!-- /BLP:TITLE -->

---

<!-- BLP:1 -->
## §1: Planteamiento del Problema

El parser Python oficial de CODEC-CORTEX (codec_cortex/parser.py) solo procesa 23 de los 40 casos del corpus F4. La implementación Go (codec-cortex-go/cortex/parser.go) procesa los 40 sin errores. Los 17 casos fallidos en Python incluyen cuerpos multilínea, namespaces calificados, cadenas escapadas, y combinaciones de shapes — todos manejados correctamente por Go.

**Evidencia:**
- Go procesa 40/40 del corpus F4 (verificado el 2026-07-17)
- Python falla en 17/40, incluyendo H008-H010 (multilinea), H016/H034 (namespaces), H014 (escapes), H028 (combinacion), H025-H027 (cuerpo/bloque), H032, H036, H038-H039
- La API publica de parse_cortex() es la que recibira quien haga `pip install codec-cortex`

**Impacto de no resolverlo:**
- El parser oficial (Python) se publica en PyPI con solo 57% de cobertura del corpus
- Un usuario externo encontrara errores en casos de uso reales
<!-- /BLP:1 -->

<!-- BLP:2 -->
## §2: Objetivo

Robustecer el parser Python (codec_cortex/parser.py) para procesar 40/40 casos del corpus F4, tomando como referencia la implementacion Go existente. Sin cambiar la API publica. Sin cambios al formato CORTEX.
<!-- /BLP:2 -->

<!-- BLP:8 -->
## §8: Diseño Técnico

El parser Python actual (scalars.py + parser.py) tiene una estructura basada en el codigo del revisor externo. Las correcciones se centran en funciones especificas sin reestructurar el modulo:

1. **Multilinea (cuerpo/bloque):** La funcion _parse_cuerpo_payload() y _parse_bloque_payload() en parser.py deben manejar correctamente la deteccion de llave de cierre incluso cuando hay llaves anidadas en el contenido. Go usa un contador de profundidad de llaves — misma logica en Python.

2. **Namespaces:** La funcion _parse_idea_head() en parser.py debe reconocer el patron ALIAS::SIGILO:nombre y separar correctamente namespace, sigilo y nombre. Go usa split con regex.

3. **Escapes:** La funcion parse_string_literal() en scalars.py debe interpretar backslash-escapes (\\n, \\t, \\") en lugar de tratarlos como literales.
<!-- /BLP:8 -->
