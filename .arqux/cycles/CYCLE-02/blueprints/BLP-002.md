# BLP-002: Crear documentación pública de CODEC-CORTEX que permita a un externo entender el proyecto en 5 minutos, instalar el paquete y producir CORTEX válido sin leer la especificación completa
<!-- /BLP:TITLE -->

---

<!-- BLP:1 -->
## §1: Planteamiento del Problema

CODEC-CORTEX no tiene documentación pública apta para un externo. El README actual es escueto (34 líneas). No hay guía rápida, ni página de inicio, ni ejemplo de uso desde cero. Un desarrollador que encuentre el proyecto en PyPI o GitHub no puede entender qué es CORTEX, cómo usarlo, ni por qué debería importarle — sin leer la especificación completa de 30+ secciones.

**Evidencia:**
- README.md tiene 34 líneas, sin ejemplos de código
- No hay docs/index.md ni guía de inicio rápido
- El skill autocontenido existe pero no está enlazado desde README
- No hay referencias cruzadas entre el skill, la spec y las implementaciones

**Impacto de no resolverlo:**
- La publicación en PyPI y GitHub no tendrá documentación de entrada
- Adopción externa bloqueada por falta de contexto inicial
<!-- /BLP:1 -->

<!-- BLP:2 -->
## §2: Objetivo

- Crear docs/authoring/ con:
  - docs/authoring/custom-sigils.md — guia para crear sigilos personalizados (declaracion en $0, types, fields, enums, schemas)
  - docs/authoring/hcortex-templates.md — guia para crear plantillas HCORTEX humanas que compilen de vuelta a CORTEX
  - docs/authoring/index.md — pagina de entrada de authoring
- README.md renovado (vision, ejemplo CORTEX, enlaces)
- docs/QUICKSTART.md (guia 5 min con ejemplo funcional)
- docs/index.md (pagina de entrada del proyecto)
- Enlaces desde README al skill y docs de authoring
<!-- /BLP:2 -->

<!-- BLP:3 -->
## §3: Precondiciones

- BLP-001 completado (parser 40/40)
- skill/codec-cortex.skill.md existente y validado
- docs/standard/ con especificaciones completas
- 5 implementaciones funcionales
<!-- /BLP:3 -->

La documentación se escribe para un externo que no conoce CORTEX. Sin jerga interna del proyecto, sin referencias a BLPs, sin asumir contexto. Cada documento debe poder entenderse de forma independiente.
